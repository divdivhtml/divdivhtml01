div

Divineの略。

div要素は「分割する」だけなので、それ自体は意味を持ちません。したがってclassやidで命名して便宜的な意味を持たせることが望ましいです。
また、今後HTML5のマークアップにおいては構造を明示した新しい要素への置き換えが求められ、利用は最小限にとどめるべきと提言されています。

<div>HTML5タグリファレンス

dl

HTML4.01では Definition List「定義リスト」

「<DT>とは<DD>である」ため、<DT>と<DD>は一対一で用いないと構文エラーとなります。

<DL>HTMLタグリファレンス

HTML5以降は description List「説明リスト」

「<dt>〜の説明が<dd>〜」となり、<dt>と<dd>は必ずしも一対一である必要はなくなりましたが、どちらかが一つもなくて組み合わせが成立しない場合は構文エラーとなります。

<dl>HTML5タグリファレンス

dt

HTML4.01では definition term（定義語）

<DT>HTMLタグリファレンス

HTML5以降は description term（説明される言葉）

<dt>HTML5タグリファレンス

dd

HTML4.01では definition or value in a definition list.（定義リストの定義または値）

<DD>HTMLタグリファレンス

HTML5以降は description or value in a description list.（説明リストの説明または値）

<dd>HTML5タグリファレンス

span

span要素は「文や画像の範囲を指定する」だけなので、それ自体は意味を持ちません。したがってclassやidで命名して便宜的な意味を持たせることが望ましいです。
また、HTML5においてブロック要素／インライン要素の分類はなくなりましたが、div要素との使い分けについては変化はありません。
